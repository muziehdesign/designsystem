Button

Use a button to allow user to perform an action.

Variants

Each button variant serves a specific purpose, they should be used accordingly.

Primary

Use `primary` for primary action of a container. While not every container requires a primary button, it should only appear once per container (not including application header, modal, or side panel).

Secondary/Default

Default button to use for actions. They can be used alone or as a secondary button along with a primary button.

Danger

Use a `danger` button for the primary action when it would causes destruction on the data. This is usually seen in confirmation modals.

Warning

Use a `warning` button to warn the user when the associated action would cause significant changes. This is usually seen in modals.

States

Disabled

Disabled buttons can cause accessibility problems and should be avoided.

Loading

Button group

Use .button-list when displaying a group of buttons to enforce consistent button spacing.

Button with icon

Use button with an icon when the visual presentation helps conveying meaning quicker and attention is needed for the action.

Best practices

  • Use concise active verbs or phrases that clearly indicate action
  • Use sentence-case captitalization, e.g. Primary button and not Primary Button
