Alerts

Use alerts to display an inline message to the user. Illustration of idea only. Styles and color selection need to be done.

Info

Use an info alert to provide important information. This is the default to use.

A simple alert with text.

Success

Use a success alert to inform about successful system events or user actions.

Error

Use an error alert for critical errors, whether that be caused by user or system.

A simple alert with text.

Warning

Use a warning alert to warn possible issue that requires attention.

Account was locked due to excessive login attempts.

Anatomy

  1. Icon and background color

    Icon and background color help user identify alert type.

  2. Title

    Title is clear and concise text that indicate purpose of alert message. Required.

  3. Body (optional)

    Body should be concise and informative text that describes the issue and actions user needs to take.

  4. Footer (optional)

    Footer should contain One or more link buttons to enable the user to act on the issue. Use either <a> or <button> with link appearance.
