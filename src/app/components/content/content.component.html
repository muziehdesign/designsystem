Content

Primary content

Primary content is the essential of the current page. It helps the user quickly identify what's being looked at. On a detail page, it typically needs to be loaded before anything else because other requests depend on it. Primary content is what should be used to determine status of the page -- success or not found.

Secondary content

Secondary content is information related to the primary content. Loading of secondary content can be deferred (display page even when secondary content is still being fetched) to display information to user sooner.

User should be given an option to reload in case of failure if it is something critical to the page experience. Otherwise, loading error can be ignored.
